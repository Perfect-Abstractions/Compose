<DocSubtitle>
Access and manage ERC-1155 token data within a diamond
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes ERC-1155 data query functions via diamond proxy.
- Reads token data directly from shared diamond storage.
- Includes `exportSelectors` for diamond registration automation.
- No state-changing logic; functions are `view` or `pure`.
</Callout>

## Overview

This facet provides external read-only access to ERC-1155 token balances and approval status. It routes calls through the diamond proxy, leveraging shared diamond storage. Developers add this facet to a diamond to expose ERC-1155 token data in a composable and upgradeable manner.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `ERC1155DataFacet` selectors are correctly registered within the diamond's facet registry.
- Call facet functions through the diamond proxy address to ensure proper routing and access control.
- Verify that other facets do not interfere with the storage layout used by `ERC1155Storage`.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
This facet contains only view and pure functions, posing no direct reentrancy risk. Input validation for array lengths in `balanceOfBatch` is handled by the `ERC1155InvalidArrayLength` error. Ensure proper access control is enforced by the diamond proxy for any functions that might interact with or depend on the data provided by this facet.
</Callout>
