<DocSubtitle>
Approves token spending on behalf of an owner
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes the `approve` function for ERC-20 token spending approvals.
- Utilizes diamond storage for managing token approval state.
- Includes `getStorage` for internal access to storage structure.
- Provides `exportSelectors` for diamond selector registration.
</Callout>

## Overview

This facet implements the ERC-20 `approve` function within a Compose diamond. It allows token owners to grant permission to a spender to withdraw tokens from their account. Calls are routed through the diamond proxy, ensuring composability and upgradeability. Developers integrate this facet to enable token approval functionality for their diamond.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `ERC20ApproveFacet` is initialized with correct storage slot references during diamond deployment.
- Verify that the `approve` function is protected by appropriate access control if the diamond requires it.
- Use `exportSelectors` during diamond deployment to register the facet's functions.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `approve` function requires careful input validation for the `_spender` address to prevent unexpected approvals. The `ERC20InvalidSpender` error is emitted if the spender address is invalid (e.g., zero address). Follow standard Solidity security practices for token transfers and approvals.
</Callout>
