Destroys ERC-721 tokens and removes them from enumeration tracking.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Functions are `internal`, intended for use within other facets.
- Manages token destruction and enumeration state within diamond storage.
- Utilizes inline assembly to access shared storage via `STORAGE_POSITION`.
- Does not perform approval checks; relies on calling facets for this logic.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides internal functionality to burn ERC-721 tokens, ensuring they are also removed from enumeration tracking. By calling `burn`, facets can manage token lifecycle within the diamond's shared storage. Changes to token existence and enumeration are immediately visible to all facets interacting with the ERC721Storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure approval checks are performed by the calling facet before invoking `burn`.
- Handle the `ERC721NonexistentToken` error, which may be emitted if the token does not exist.
- Verify the diamond storage layout for `STORAGE_POSITION` is correctly set and compatible when upgrading facets.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage at the slot identified by `keccak256("erc721.enumerable")`. The `burn` function modifies the state of the `ERC721EnumerableStorage` struct, directly impacting token existence and enumeration order. These changes are immediately visible to any facet that reads from the same storage slot.
</Callout>
