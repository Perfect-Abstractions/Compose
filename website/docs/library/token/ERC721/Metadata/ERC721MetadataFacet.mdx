ERC-721 token metadata management
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external view functions for token metadata retrieval.
- Accesses shared diamond storage for name, symbol, and base URI.
- Provides `exportSelectors` for easy diamond integration.
- Self-contained facet adhering to Compose standards.
</Callout>

## Overview

This facet provides external functions for retrieving ERC-721 token metadata within a Compose diamond. It accesses shared diamond storage to return the token collection's name, symbol, and individual token URIs. Developers integrate this facet to expose standard ERC-721 metadata properties while leveraging the diamond's upgradeability.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `ERC721MetadataFacet` is registered with the diamond's diamond cut mechanism.
- Call metadata functions (`name`, `symbol`, `tokenURI`) through the diamond proxy address.
- Verify that the `STORAGE_POSITION` for metadata is correctly initialized by the diamond.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All functions are `view` and do not modify state, thus posing no reentrancy risk. Input validation for `tokenURI` is handled by underlying ERC-721 logic, which is assumed to revert on non-existent tokens or invalid owner states. Ensure proper access control is managed at the diamond level for functions that might be added in the future which interact with this storage.
</Callout>
