Internal logic for ERC-721 token management.

<Callout type="info" title="Key Features">
- Provides internal utility functions for ERC-721 compliant token operations.
- Directly interacts with diamond storage for token state management.
- Supports minting, burning, and transferring tokens with robust validation.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC721Mod provides essential internal functions for managing ERC-721 tokens within a Compose diamond. It handles core operations like minting, burning, and transferring tokens, ensuring state integrity through diamond storage access. This module enables facets to implement ERC-721 compliance safely and efficiently.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IERC721Mod } from "@compose/modules/ERC721Mod.sol";
import {IERC721Storage } from "@compose/storage/ERC721Storage.sol";

contract MyERC721Facet {
    address constant ERC721_STORAGE_SLOT = 0x1234567890; // Example slot

    function mintToken(address _to, uint256 _tokenId) external {
        IERC721Storage erc721Storage = IERC721Storage(ERC721_STORAGE_SLOT);
        IERC721Mod(address(this)).mint(erc721Storage, _to, _tokenId);
    }

    function transferToken(address _from, address _to, uint256 _tokenId) external {
        IERC721Storage erc721Storage = IERC721Storage(ERC721_STORAGE_SLOT);
        IERC721Mod(address(this)).transferFrom(erc721Storage, _from, _to, _tokenId);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the ERC721Storage struct is correctly initialized and accessible via its designated slot.
- Always validate `_to` addresses to prevent minting to the zero address.
- Handle potential `ERC721IncorrectOwner`, `ERC721InsufficientApproval`, and `ERC721NonexistentToken` errors appropriately in facet logic.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module accesses ERC721 state via the `ERC721Storage` struct, which is expected to reside at a predefined storage slot within the diamond. Facets using this module must pass the `ERC721Storage` instance to the module functions. The `getStorage` function can be used by facets to retrieve the storage layout. Ensure that the storage slot for `ERC721Storage` is correctly configured and that no other facets or modules overwrite this critical state.
</Callout>
