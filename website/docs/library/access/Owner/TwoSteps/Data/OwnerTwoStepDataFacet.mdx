<DocSubtitle>
Manages pending owner state for two-step ownership transfers
</DocSubtitle>

<Callout type="info" title="Key Features">
- Exposes external `pendingOwner` view function for querying.
- Provides internal `getStorage` function to access the raw storage structure.
- `exportSelectors` function aids in diamond facet management.
- Operates on shared diamond storage via `STORAGE_POSITION`.
</Callout>

## Overview

This facet provides access to the pending owner state within a diamond, facilitating two-step ownership transfer mechanisms. It exposes functions to retrieve the pending owner address and internal access to its storage structure. This facet works in conjunction with other owner facets and modules to manage ownership changes securely.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the `OwnerTwoStepDataFacet` is correctly initialized within the diamond, typically during deployment or upgrade.
- Access the `pendingOwner` directly through the diamond proxy address.
- Use the `exportSelectors` function for diamond upgrade and facet discovery purposes.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
All state-changing operations related to ownership are handled by other facets. This facet is read-only regarding ownership status. Standard Solidity security practices apply. Ensure correct diamond proxy routing to prevent direct facet calls that bypass diamond logic.
</Callout>
