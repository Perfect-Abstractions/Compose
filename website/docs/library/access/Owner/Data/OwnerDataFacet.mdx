<DocSubtitle>
Manage diamond owner and export facet selectors
</DocSubtitle>

<Callout type="info" title="Key Features">
- Provides an external `owner` function for owner retrieval.
- Exposes `exportSelectors` for function selector discovery.
- Accesses shared diamond storage for owner data.
- Follows ERC-2535 diamond proxy pattern.
</Callout>

## Overview

This facet provides external functions to retrieve the diamond's owner and export the facet's selectors. It accesses shared diamond storage for owner information. Developers add this facet to a diamond to expose ownership details and facilitate selector discovery.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the OwnerDataFacet is added to the diamond before attempting to call its functions.
- The `owner` function relies on ERC-173 ownership, ensure this is correctly implemented and initialized within the diamond.
- Use `exportSelectors` to programmatically discover the functions provided by this facet when integrating with other diamonds or tools.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `owner` function is a `view` function and does not pose reentrancy risks. The `exportSelectors` function is `pure` and has no state-changing implications. Access to the owner address is determined by the diamond's ERC-173 implementation.
</Callout>
