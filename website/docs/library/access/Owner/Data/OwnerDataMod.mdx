## Overview

This module provides internal functions and storage for ERC-173 contract ownership. Facets can import this module to manage ownership directly within the diamond's shared storage. This ensures ownership state is consistent across all facets and accessible via the diamond storage pattern.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `requireOwner()` before executing sensitive operations that modify ownership.
- Use `setContractOwner()` to update ownership, ensuring the caller is authorized.
- Handle `OwnerUnauthorizedAccount` and `OwnerAlreadyRenounced` errors when appropriate.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module integrates with the diamond storage pattern by using a dedicated storage slot identified by `STORAGE_POSITION` (keccak256(\"erc173.owner\")). The `OwnerStorage` struct, containing the `owner` address, is accessed via inline assembly. All functions operate on this shared storage, making ownership changes immediately visible to any facet interacting with the diamond.
</Callout>
