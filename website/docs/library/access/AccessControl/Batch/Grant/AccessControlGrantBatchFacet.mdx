<DocSubtitle>
Grants roles to multiple accounts efficiently
</DocSubtitle>

<Callout type="info" title="Key Features">
- Grants roles to multiple accounts in a single transaction.
- Emits `RoleGranted` events for each granted role.
- Reverts with `AccessControlUnauthorizedAccount` if the caller lacks permission.
- Exports its selectors for diamond registration.
</Callout>

## Overview

This facet provides an efficient way to grant a specific role to multiple accounts within a diamond. It integrates with diamond storage and leverages internal modules for role management. This facet exposes external functions for direct invocation through the diamond proxy, enabling batch role assignments in a single transaction.

## Best Practices

<Callout type="tip" title="Best Practice">
- Grant roles only to trusted accounts.
- Ensure the caller has the necessary permissions to grant roles.
- Consider using this facet during diamond initialization or upgrade processes.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `grantRoleBatch` function is protected by access control, ensuring only authorized callers can grant roles. It is critical that the caller's permissions are validated by the diamond's access control logic before this facet's function is executed. Input validation for `_accounts` is handled internally to prevent unexpected behavior. Follow standard Solidity security practices for all interactions.
</Callout>
