<DocSubtitle>
Revokes temporal roles from accounts
</DocSubtitle>

<Callout type="info" title="Key Features">
- Revokes temporal roles from specified accounts.
- Emits `TemporalRoleRevoked` event upon successful revocation.
- Reverts with `AccessControlUnauthorizedAccount` if the caller is not the role administrator.
- Exports its own selectors for diamond registration.
</Callout>

## Overview

This facet provides functionality to revoke temporal roles from accounts within a Compose diamond. It exposes the `revokeTemporalRole` function, allowing administrators to remove roles dynamically. The facet interacts with diamond storage to manage role assignments and emits `TemporalRoleRevoked` events upon successful revocation.

### revokeTemporalRole

Revokes a temporal role from an account. Only the admin of the role can revoke it. Emits a TemporalRoleRevoked event. Reverts with AccessControlUnauthorizedAccount If the caller is not the admin of the role.

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize roles and assign administrators during diamond deployment.
- Ensure the caller possesses the necessary administrative privileges before invoking `revokeTemporalRole`.
- Verify storage compatibility and selector registration when upgrading the facet.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `revokeTemporalRole` function enforces access control, ensuring only the administrator of a role can revoke it. Input validation is performed by the underlying diamond proxy and the facet's internal checks. Follow standard Solidity security practices for handling addresses and role management.
</Callout>
