<DocSubtitle>
Revokes temporal roles from accounts within a diamond.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Revokes temporal roles using `revokeTemporalRole` function.
- Emits `TemporalRoleRevoked` event upon successful revocation.
- Protects against unauthorized revocations via `AccessControlUnauthorizedAccount` error.
- Exports its selectors using `exportSelectors` for diamond interoperability.
</Callout>

## Overview

This facet provides functionality to revoke temporal roles assigned to accounts. It interacts with diamond storage to manage role revocations and emits events for off-chain monitoring. Developers integrate this facet to manage dynamic role assignments within a Compose diamond.

## Functions

### revokeTemporalRole

Revokes a temporal role from an account. Only the admin of the role can revoke it. Emits a TemporalRoleRevoked event. Reverts with AccessControlUnauthorizedAccount If the caller is not the admin of the role.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the caller has the necessary permissions to revoke the specified temporal role.
- Initialize temporal roles and their admins before attempting revocations.
- Monitor the `TemporalRoleRevoked` event for off-chain state updates.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The `revokeTemporalRole` function enforces access control, ensuring only the designated admin of a role can revoke it. Reverts with `AccessControlUnauthorizedAccount` if the caller is not the admin. Follow standard Solidity security practices for input validation.
</Callout>
