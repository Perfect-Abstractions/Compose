Renounce roles using diamond storage
</DocSubtitle>

<Callout type="info" title="Key Features">
- Internal functions for role renouncement.
- Utilizes the diamond storage pattern.
- Emits `RoleRevoked` event upon successful renouncement.
- Reverts with `AccessControlUnauthorizedSender` if caller is not the account.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

This module provides functions for accounts to renounce their roles within the diamond. It leverages the diamond storage pattern for shared access control state. By calling `renounceRole`, an account can remove itself from a specified role, ensuring that access control changes are immediately reflected across all facets using the same storage.

## Best Practices

<Callout type="tip" title="Best Practice">
- Call `renounceRole` only when an account is relinquishing its own assigned role.
- Ensure the `AccessControlStorage` struct is correctly initialized and accessible.
- Handle the `AccessControlUnauthorizedSender` error if the caller is not the intended account.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
This module interacts with diamond storage via the `STORAGE_POSITION` key, which is defined as `keccak256("compose.accesscontrol")`. The `renounceRole` function directly modifies the access control state stored at this position. Any facet reading from this storage position will immediately see the updated role assignments after a successful renouncement.
</Callout>
