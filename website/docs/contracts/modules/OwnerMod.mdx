<DocSubtitle>
ERC-173 Contract Ownership - Provides internal functions and storage layout for owner management.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Manages contract ownership according to ERC-173 standards.
- Provides a &#x60;requireOwner&#x60; modifier for access control.
- Supports ownership renouncement by transferring to &#x60;address(0)&#x60;.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The OwnerMod provides essential ERC-173 contract ownership functionality. It manages the contract owner&#x27;s address, allowing for secure ownership transfers and owner-based access control, crucial for administrative operations within a diamond.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IOwnerMod} from "@compose/modules/owner/IOwnerMod.sol";
import {OwnerStorage} from "@compose/modules/owner/OwnerStorage.sol";

contract MyOwnerFacet {
    address constant STORAGE_POSITION = 0x0; // Example storage slot

    function getOwner() external view returns (address) {
        // Access the OwnerMod storage layout
        OwnerStorage storage ownerStorage = OwnerStorage(STORAGE_POSITION);
        // Call the owner function from the module interface
        return IOwnerMod(address(ownerStorage)).owner();
    }

    function transferContractOwnership(address _newOwner) external {
        OwnerStorage storage ownerStorage = OwnerStorage(STORAGE_POSITION);
        // Call the transfer ownership function from the module interface
        IOwnerMod(address(ownerStorage)).transferOwnership(_newOwner);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Use &#x60;requireOwner()&#x60; to protect sensitive administrative functions within your facet.
- Carefully manage ownership transfers, ensuring the new owner is a trusted address.
- Implement checks for &#x60;address(0)&#x60; when transferring ownership to allow for renouncement.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The OwnerMod utilizes a specific storage slot for its &#x60;OwnerStorage&#x60; struct. Facets interacting with ownership functions must access this storage slot directly or via the &#x60;IOwnerMod&#x60; interface. The &#x60;getStorage&#x60; function can be used to retrieve a pointer to this storage if the &#x60;STORAGE_POSITION&#x60; is known. Changes to the owner address are immediately reflected across all facets interacting with the &#x60;OwnerStorage&#x60;.
</Callout>
