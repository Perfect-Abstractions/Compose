LibERC20Permit — Library for ERC-2612 Permit Logic - Thrown when a permit signature is invalid or expired.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Implements the ERC-2612 standard for off-chain signature-based token approvals.
- Utilizes a dedicated storage slot for permit-related data, ensuring composability and upgrade safety.
- Provides access to the &#x60;DOMAIN_SEPARATOR&#x60; for consistent signature generation across facets.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The ERC20PermitMod provides the necessary logic for implementing the ERC-2612 &#x27;permit&#x27; functionality within a Compose diamond. This allows users to grant allowances to other addresses via signed off-chain messages, enhancing user experience and reducing gas costs for frequent allowance updates.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {LibERC20Permit} from "@compose-protocol/diamond-contracts/contracts/module/erc20/LibERC20Permit.sol";
import {IERC20Permit} from "@compose-protocol/diamond-contracts/contracts/interface/IERC20Permit.sol";

contract MyERC20Facet {
    using LibERC20Permit for LibERC20Permit.PermitStorage;

    function permit(IERC20Permit._Permit calldata _permit, bytes32 _signature) external {
        LibERC20Permit.PermitStorage storage ps = LibERC20Permit.getPermitStorage();
        address owner = ps.permit(_permit, _signature);
        // The permit function emits the Approval event.
        // If this facet needs to emit an Approval event, it must be done here.
        // For example: emit Approval(owner, _permit.spender, _permit.value);
    }

    function DOMAIN_SEPARATOR() external view returns (bytes32) {
        return LibERC20Permit.DOMAIN_SEPARATOR();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the &#x60;permit&#x60; function is called within a facet that correctly emits the &#x60;Approval&#x60; event as specified by ERC-20 standards, as &#x60;LibERC20Permit.permit&#x60; itself does not emit it.
- Always verify the &#x60;DOMAIN_SEPARATOR&#x60; is correctly configured and matches the chain ID and contract address when implementing off-chain permit signing.
- Treat signature validation as a critical security step; ensure all parameters for &#x60;LibERC20Permit.permit&#x60; are accurately passed from the signed message.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The ERC20PermitMod interacts with the diamond&#x27;s storage through the &#x60;LibERC20Permit.PermitStorage&#x60; struct. Facets integrating this module must retrieve this storage using &#x60;LibERC20Permit.getPermitStorage()&#x60;. The &#x60;permit&#x60; function within the library validates signatures and updates the allowance, but the responsibility of emitting the &#x60;Approval&#x60; event lies with the calling facet, ensuring adherence to the ERC-20 standard&#x27;s event emission requirements.
</Callout>
