Emitted when the admin role for a role is changed.
</DocSubtitle>

<Callout type="info" title="Key Features">
- Role-based access control: Define and manage distinct roles with granular permissions.
- Permission management functions: &#x60;grantRole&#x60;, &#x60;revokeRole&#x60;, and &#x60;setRoleAdmin&#x60; provide flexible control over role assignments and administration.
- Explicit role checking: &#x60;hasRole&#x60; and &#x60;requireRole&#x60; allow for clear and auditable permission checks within facets.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The AccessControlMod module provides a robust role-based access control system for Compose diamonds. It enables granular permission management by allowing roles to be granted, revoked, and checked, ensuring that only authorized accounts can perform sensitive operations. This is crucial for maintaining the security and integrity of diamond applications.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IAccessControl} from "@compose/diamond-contracts/contracts/modules/access/IAccessControlMod.sol";

contract AccessControlUserFacet {
    IAccessControl internal accessControl;

    // Assume accessControl is initialized in an initializer function
    // and points to the AccessControlMod facet address.

    bytes32 public constant ADMIN_ROLE = keccak256("ADMIN_ROLE");
    bytes32 public constant MINTER_ROLE = keccak256("MINTER_ROLE");

    function grantMinterRole(address _account) external {
        // Ensure the caller has the authority to grant roles
        accessControl.requireRole(_account, ADMIN_ROLE);
        accessControl.grantRole(MINTER_ROLE, _account);
    }

    function hasMinterRole(address _account) external view returns (bool) {
        return accessControl.hasRole(MINTER_ROLE, _account);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always use &#x60;requireRole&#x60; for internal checks within facets to enforce access control before executing critical logic.
- Manage role administration carefully. Only grant the &#x60;ADMIN_ROLE&#x60; to trusted accounts, and use &#x60;setRoleAdmin&#x60; to define clear administrative hierarchies for other roles.
- When upgrading the diamond, ensure the AccessControlMod facet is properly updated or re-initialized if its storage layout changes, to maintain consistent access control.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The AccessControlMod module manages its state within the diamond&#x27;s storage. Facets interact with it through the &#x60;IAccessControl&#x60; interface. Changes to roles and role admins made via the AccessControlMod functions are immediately reflected and visible to all facets interacting with the module. Ensure the &#x60;AccessControlMod&#x60; facet is deployed and its address is correctly registered in the diamond&#x27;s facet registry for other facets to interact with it. The module relies on the diamond&#x27;s storage pattern for state persistence.
</Callout>
