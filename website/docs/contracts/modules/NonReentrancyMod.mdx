<Callout type="info" title="Key Features">
- Prevents reentrancy by maintaining an internal non-reentrancy lock state.
- Provides explicit &#x60;enter&#x60; and &#x60;exit&#x60; functions for clear control flow.
- Designed for integration within Compose diamond facets.
</Callout>

<Callout type="info" title="Module Usage">
This module provides internal functions for use in your custom facets. Import it to access shared logic and storage.
</Callout>

## Overview

The NonReentrancyMod module provides essential utilities for preventing reentrancy attacks within your diamond. By utilizing its &#x60;enter&#x60; and &#x60;exit&#x60; functions, facets can enforce strict execution order, ensuring that critical operations are not interrupted by recursive calls, thereby enhancing the security and integrity of your diamond&#x27;s state.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {LibNonReentrancy} from "@compose/modules/NonReentrancyMod/LibNonReentrancy.sol";

contract MyFacet {
    using LibNonReentrancy for uint256;

    uint256 public _nonReentrancyState; // Example state protected by non-reentrancy

    /**
     * @notice Performs a protected operation.
     */
    function protectedOperation() external {
        // Enter the non-reentrancy guard
        _nonReentrancyState = _nonReentrancyState.enter();

        // Perform critical state changes here...
        // For example, emit an event, update storage, etc.

        // Exit the non-reentrancy guard
        _nonReentrancyState = _nonReentrancyState.exit();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Always pair &#x60;enter()&#x60; with a corresponding &#x60;exit()&#x60; call to ensure the guard is properly released, even in error scenarios (e.g., using &#x60;try/catch&#x60; or &#x60;require&#x60; statements that might revert before &#x60;exit()&#x60;).
- Use &#x60;LibNonReentrancy.enter()&#x60; at the very beginning of a function and &#x60;LibNonReentrancy.exit()&#x60; at the very end to provide the broadest protection.
- Ensure the variable used with &#x60;LibNonReentrancy&#x60; is appropriately scoped and managed within the facet&#x27;s storage.
</Callout>

## Integration Notes

<Callout type="success" title="Shared Storage">
The NonReentrancyMod is intended to be used as a library within facets. Facets will typically declare a storage variable (e.g., &#x60;uint256 _nonReentrancyState&#x60;) and use the &#x60;LibNonReentrancy&#x60; functions with this variable to manage the reentrancy lock. The state managed by this library is local to the facet&#x27;s execution context and does not directly interact with or modify the diamond&#x27;s shared storage slots. Each facet using this module is responsible for its own reentrancy protection.
</Callout>
