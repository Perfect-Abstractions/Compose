Add&#x3D;0, Replace&#x3D;1, Remove&#x3D;2
</DocSubtitle>

<Callout type="info" title="Key Features">
- Supports adding new functions by linking selectors to new facet addresses.
- Enables updating existing functionality by replacing old facet addresses with new ones for specific selectors.
- Allows removal of functions by unlinking selectors, effectively disabling them.
- Facilitates atomic upgrades by allowing a function to be executed immediately after the cut.
</Callout>

## Overview

The DiamondCutFacet provides the core functionality for upgrading and managing the functions within a Compose diamond. It allows adding, replacing, and removing facet functions, enabling dynamic contract logic updates and feature extensibility.

## Best Practices

<Callout type="tip" title="Best Practice">
- Ensure the &#x60;diamondCut&#x60; function is only callable by authorized addresses (e.g., an owner or a governance contract) to prevent unauthorized upgrades.
- Carefully manage the mapping of function selectors to facet addresses during upgrades to avoid breaking existing functionality or introducing vulnerabilities.
- When replacing or removing functions, consider the impact on dependent facets and external contracts interacting with the diamond.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Access to &#x60;diamondCut&#x60;, &#x60;addFunctions&#x60;, &#x60;replaceFunctions&#x60;, and &#x60;removeFunctions&#x60; must be strictly controlled. Unauthorized calls can lead to the diamond&#x27;s functionality being compromised. Ensure that the &#x60;diamondCut&#x60; function is not susceptible to reentrancy if it includes an optional &#x60;delegatecall&#x60;. Input validation on function selectors and facet addresses is crucial.
</Callout>
