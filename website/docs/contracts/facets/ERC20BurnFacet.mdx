<DocSubtitle>
Contract documentation for ERC20BurnFacet
</DocSubtitle>

<Callout type="info" title="Key Features">
- Allows for the reduction of total token supply by destroying tokens.
- Supports burning tokens directly from the caller&#x27;s balance.
- Enables burning tokens from another account, contingent on an approved allowance.
</Callout>

## Overview

The ERC20BurnFacet provides functionality to destroy ERC20 tokens within a Compose diamond. It enables users to burn their own tokens or burn tokens from other accounts if an allowance is set, facilitating token supply reduction and management.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IDiamondCut} from "@compose/diamond-solidity/src/diamond/IDiamondCut.sol";
import {IERC20BurnFacet} from "@compose/diamond-solidity/src/facets/erc20/IERC20BurnFacet.sol";

contract Deployer {
    address diamondAddress;

    function deploy() public {
        // ... deployment logic to create the diamond proxy ...
        diamondAddress = address(0xYourDiamondProxyAddress);

        // Add ERC20BurnFacet
        address erc20BurnFacetAddress = address(new ERC20BurnFacet());
        bytes32[] memory selectors = new bytes32[](3);
        selectors[0] = IERC20BurnFacet.getStorage.selector;
        selectors[1] = IERC20BurnFacet.burn.selector;
        selectors[2] = IERC20BurnFacet.burnFrom.selector;

        bytes[] memory functionSignatures = new bytes[](3);
        functionSignatures[0] = abi.encodeWithSignature("getStorage() returns (ERC20Storage storage)");
        functionSignatures[1] = abi.encodeWithSignature("burn(uint256 _amount)");
        functionSignatures[2] = abi.encodeWithSignature("burnFrom(address _from, uint256 _amount)");

        IDiamondCut(diamondAddress).diamondCut(
            IDiamondCut.FacetCut[]
                (
                    {
                        facetAddress: erc20BurnFacetAddress,
                        action: IDiamondCut.Action.ADD,
                        functionSelectors: selectors
                    }
                ),
            address(0),
            ""
        );
    }

    function burnMyTokens() public {
        IERC20BurnFacet(diamondAddress).burn(100 * 1 ether);
    }

    function burnAllowanceTokens(address _spender, uint256 _amount) public {
        IERC20BurnFacet(diamondAddress).burnFrom(_spender, _amount);
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the ERC20BurnFacet within your diamond&#x27;s deployment script, ensuring the correct selectors are mapped.
- When calling &#x60;burnFrom&#x60;, ensure an allowance has been previously set for the caller by the &#x60;_from&#x60; address.
- The &#x60;burn&#x60; and &#x60;burnFrom&#x60; functions correctly emit a &#x60;Transfer&#x60; event to the zero address, signaling token destruction.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The &#x60;burnFrom&#x60; function relies on the ERC20 &#x60;allowance&#x60; mechanism. Ensure that the &#x60;allowance&#x60; is managed securely and that users understand the implications of granting allowances. No reentrancy risks are present as the functions do not make external calls. Input validation for &#x60;_amount&#x60; should be handled by the caller or the ERC20 storage logic to prevent underflows or overflows.
</Callout>
