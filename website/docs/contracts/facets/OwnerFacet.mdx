<Callout type="info" title="Key Features">
- Manages the single owner address for the diamond contract.
- Supports transferring ownership to a new address.
- Allows for complete renunciation of ownership.
</Callout>

## Overview

The OwnerFacet provides essential ownership management for a Compose diamond. It allows for retrieving the current owner, transferring ownership to a new address, and renouncing ownership entirely. This facet is crucial for controlling administrative actions and ensuring secure contract governance.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IOwnerFacet} from "@compose-protocol/diamond/contracts/facets/Owner/IOwnerFacet.sol";

contract OwnerConsumer {
    IOwnerFacet private immutable _ownerFacet;

    constructor(address _diamondAddress) {
        _ownerFacet = IOwnerFacet(_diamondAddress);
    }

    function getDiamondOwner() external view returns (address) {
        return _ownerFacet.owner();
    }

    function transferDiamondOwnership(address _newOwner) external {
        _ownerFacet.transferOwnership(_newOwner);
    }

    function renounceDiamondOwnership() external {
        _ownerFacet.renounceOwnership();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize the diamond with the OwnerFacet to establish initial ownership.
- Use &#x60;transferOwnership&#x60; for controlled transitions of administrative control.
- Ensure the &#x60;owner&#x60; address has appropriate permissions for critical diamond functions.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
The &#x60;transferOwnership&#x60; function can be used to set the new owner to &#x60;address(0)&#x60;, effectively renouncing ownership. This action is irreversible. Access to &#x60;transferOwnership&#x60; and &#x60;renounceOwnership&#x60; is restricted to the current owner.
</Callout>
