<Callout type="info" title="Key Features">
- Secure two-step ownership transfer mechanism.
- Explicit &#x60;acceptOwnership&#x60; call prevents accidental ownership changes.
- Functions &#x60;owner()&#x60;, &#x60;pendingOwner()&#x60;, &#x60;transferOwnership()&#x60;, &#x60;acceptOwnership()&#x60;, and &#x60;renounceOwnership()&#x60; provide full ownership management capabilities.
</Callout>

## Overview

The OwnerTwoStepsFacet manages contract ownership through a two-step verification process. It allows the current owner to initiate a transfer to a new owner, who must then explicitly accept the ownership, enhancing security against accidental or malicious ownership changes.

## Usage Example

<ExpandableCode language="solidity" maxLines={20} title="Example Implementation">
{`pragma solidity ^0.8.30;

import {IOwnerTwoStepsFacet} from "../interfaces/IOwnerTwoStepsFacet.sol";

contract OwnerDeployer {
    address immutable diamondAddress;

    constructor(address _diamondAddress) {
        diamondAddress = _diamondAddress;
    }

    function transferOwnershipToNewOwner(address _newOwner) public {
        IOwnerTwoStepsFacet(diamondAddress).transferOwnership(_newOwner);
    }

    function acceptOwnershipFromCurrentOwner() public {
        IOwnerTwoStepsFacet(diamondAddress).acceptOwnership();
    }

    function renounceContractOwnership() public {
        IOwnerTwoStepsFacet(diamondAddress).renounceOwnership();
    }
}`}
</ExpandableCode>

## Best Practices

<Callout type="tip" title="Best Practice">
- Initialize ownership by calling &#x60;transferOwnership&#x60; with the desired address, followed by the new owner calling &#x60;acceptOwnership&#x60;.
- Use &#x60;owner()&#x60; and &#x60;pendingOwner()&#x60; to track the current and awaiting owner.
- Store &#x60;OWNER_STORAGE_POSITION&#x60; and &#x60;PENDING_OWNER_STORAGE_POSITION&#x60; constants within your diamond deployment scripts or configuration for consistent slot access.
</Callout>

## Security Considerations

<Callout type="warning" title="Security">
Ownership transfer functions (&#x60;transferOwnership&#x60;, &#x60;acceptOwnership&#x60;, &#x60;renounceOwnership&#x60;) are typically protected by access control mechanisms within the diamond&#x27;s governance framework. Ensure that only authorized entities can call these functions. &#x60;transferOwnership&#x60; sets a pending owner, which is only updated to the actual owner upon a successful &#x60;acceptOwnership&#x60; call by the pending owner. Direct access to storage slots via &#x60;getOwnerStorage&#x60; and &#x60;getPendingOwnerStorage&#x60; requires careful handling to avoid unintended state modifications.
</Callout>
